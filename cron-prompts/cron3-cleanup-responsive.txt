--- Fix 3: Responsive portrait/spread — MOST IMPORTANT FIX (Bug H3) ---

Currently usePortrait={true} is hardcoded in HTMLFlipBook. On desktop, the book shows one page at a time (like mobile) instead of a beautiful two-page spread. This needs to be responsive.

Read the full book-viewer/index.tsx carefully before editing. Then:

7. Near the top of the BookViewer component function (after the existing useState declarations), add:
   const [isPortrait, setIsPortrait] = useState(true);

8. Add a useEffect to listen for viewport changes:
   useEffect(() => {
       const mq = window.matchMedia('(min-width: 768px)');
       const handler = (e: MediaQueryListEvent) => setIsPortrait(!e.matches);
       setIsPortrait(!mq.matches); // Set initial value
       mq.addEventListener('change', handler);
       return () => mq.removeEventListener('change', handler);
   }, []);

9. Find the HTMLFlipBook component (around line 330-350). Change:
   usePortrait={true}

   usePortrait={isPortrait}

10. Now fix the dot navigation math. Find the dot navigation section (after the HTMLFlipBook closing tag). The current code has:

    const dotCount = Math.ceil(pages.length / 2);

    The problem: in portrait mode, react-pageflip reports page numbers as individual pages (0,1,2,3...). In spread mode, it also increments page numbers individually but pages are shown in pairs.

    Replace those two lines with:
    // Portrait (mobile): one page per view — dotCount = total pages, activeDot = currentPage
    // Spread (desktop): pages pair up — dotCount = half of total, activeDot accounts for pairs
    const dotCount = isPortrait ? pages.length : Math.ceil(pages.length / 2);
    const activeDot = isPortrait
        ? currentPage
        : currentPage === 0 ? 0 : Math.ceil(currentPage / 2);

    Note: This IIFE is inside showNavigation && (() => { ... })() — edit the two lines inside it.
